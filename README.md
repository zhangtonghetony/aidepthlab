# aidepthlab
aidepthlab 是我的个人 AI 项目集成实验室。本项目旨在探索深度学习模型和RAG/Agent在金融金融（本人主修金融专业）的落地应用，涵盖了从底层算法实现到 Web 端全栈部署的完整链路。
# 项目概览
本项目目前集成以下四个核心 AI 子项目，通过统一的 Web 界面进行交互演示：

1. 金融文本汉译英系统 (Finance-Translation)
核心算法：基于 GRU (Gated Recurrent Unit) 与 Attention (注意力机制) 的 Seq2Seq 架构。

项目亮点：
针对金融、商业语料进行专项训练，精准处理“贸易顺差”、“平仓”等专业术语。

调用开源大模型API批量生成双语语料，保障翻译准确性和语料获取效率。

采用teacher-forcing并动态调整比率，平衡模型泛化能力和收敛速度。

技术细节：实现了 Encoder-Decoder 架构，支持 Greedy Search 推理。

2. 金融评论风险等级判断 (Finance-Risk-Assessment)
核心算法：基于 BERT 预训练模型进行下游任务微调（Fine-tuning）。

项目亮点：

可解释性：引入 LIME 局部线性近似分析，可视化模型决策的关键特征词。

不平衡优化：采用带权重的 CrossEntropyLoss，有效提升高风险样本召回率。

技术细节：采用分层解冻策略（Layer Unfreezing），优化特定语义分布。

3. 金融领域 RAG+Agent 综合系统 (RAG-Agent-System)
核心算法：基于 LangChain 框架构建的检索增强生成（RAG）与智能代理（Agent）架构。

项目亮点：

自动化报告生成：集成了具备任务规划能力的 Agent，可根据检索内容自动撰写并导出可下载的金融分析报告。

异步处理机制：利用 threading 库实现文档向量化的后台独立运行，提升用户交互响应速度。

工程化安全设计：引入 Session 级别的权限控制与轻量级历史消息存储，解决越权漏洞并实现上下文感知的智能问答。

技术细节：使用 ChromaDB 进行向量空间检索，通过动态提示词（Prompt Injection）切换实现多场景适配。

4.  金融领域文本摘要生成 (Finance-Summary)
核心算法：基于标准 Transformer 编码器-解码器（Encoder-Decoder）架构的生成式摘要模型。

项目亮点：

多粒度摘要提取：支持针对长篇研报、公告的关键词句提取及高度压缩的生成式摘要。

推理可控性：通过调节解码阶段的 Temperature（采样温度），实现从“保守准确”到“创意丰富”的输出风格切换。

优化训练策略：在训练中应用了 Warmup（预热） 与 余弦退火学习率调度，显著提升了模型在垂直语料上的收敛速度。

技术细节：支持中英文混合文本处理，采用结构化输出设计，便于后续业务系统对接。

开发者说明 ：
算力限制：受限于个人开发环境算力约束，模型训练迭代次数有限。

语料覆盖：语料收集广度尚在完善中，本项目目前仅作为 PoC 与算法逻辑演示使用，不代表工业化部署的最优效果。

资源消耗：在 2核2G 等低配云服务器部署时，建议开启 Swap 交换空间 以确保 BERT 模型推理时的稳定性。
